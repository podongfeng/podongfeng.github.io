【计蒜客】跳跃游戏

今天继续介绍2道贪心算法的题目。

跳跃游戏

给定一个非负整数数组，假定你的初始位置为数组第一个下标。
数组中的每个元素代表你在那个位置能够跳跃的最大长度。
请确认你是否能够跳跃到数组的最后一个下标。
例如：
A = [2,3,1,1,4],
return true.
A = [3,2,1,0,4],
return false.

思路

我们可以从后往前看：如果要到达最后1个元素，可以先考虑倒数第2个元素，如果从倒数第2个元素可以到达最后1个元素，那么我们的问题就变成了一个len-1长度数组的问题。
根据上面简单的思考，我们的方法是这样的，用一个指针min记录可以到达最后1个元素的最小下标，初始的时候，min=len-1;从倒数第2个元素开始向前遍历，如果下标i+num[i]>min,证明从下标为i的元素可以到达最后1个元素（通过min指向的元素），这时我们就把min指向i,只要可以打到i，就可以到达最后一个元素。等到遍历完成的时候，如果min指向了第1个元素，那么就是可以到达，不然就是不可到达。

代码

然后我们来这一题的进阶版：
给定一个非负整数数组，假定你的初始位置为数组第一个下标。
数组中的每个元素代表你在那个位置能够跳跃的最大长度。
你的目标是到达最后一个下标，并且使用最少的跳跃次数。
例如：
A = [2,3,1,1,4], 到达最后一个下标的最少跳跃次数为2.（先跳跃1步，从下标0到1,然后跳跃3步，到达最后一个下标。一共两次）

思路

跟上面的题目稍有不同，这次不只要可达，而且要用最少的跳跃次数。
这其实已经是动态规划的范畴了。新建一个长度为len的数组min，保存从当前位置跳跃到最后1个元素需要的最少次数，如果不能到达，记为-1。初始数组min全部为-1，当然，min[len-1]=0，到达自身不需要跳跃。
然后从后往前遍历，遍历到下标为i的时候，内部循环遍历i+1到i+numi,取得从i开始跳跃1步达到的点，这些点需要满足min[j]>=0，就是可达最后1个元素，然后选出其中最小的，那么min[i]就是这个最小值+1.
